Core Functions

re.search(p, s)Scan s, return first match anywhere as a Match object, or None.
re.match(p, s)Match only at the start of the string.
re.fullmatch(p, s)Match only if the entire string matches.
re.findall(p, s)Return list of all non-overlapping matches (strings, or tuples if groups exist).
re.finditer(p, s)Return an iterator of Match objects for all matches.
re.sub(p, r, s)Replace matches of p in s with r; returns new string.
re.subn(p, r, s)Same as sub, but returns (new_string, count_of_subs).
re.split(p, s)Split s by occurrences of pattern p.
re.compile(p)Pre-compile pattern into a reusable Pattern object.
re.escape(s)Escape all regex special characters in literal string s.
import re
re.findall(r"\d+", "a1 b22 c333")
# ['1', '22', '333']

Match Object Methods

.group()Whole match (same as .group(0)).
.group(n)Text of capture group n (1-indexed).
.group('name')Text of a named capture group.
.groups()Tuple of all numbered groups.
.groupdict()Dict of all named groups.
.start([n])Start index of match (or group n).
.end([n])End index of match (or group n).
.span([n])Tuple (start, end).
m = re.search(r"(?P<y>\d{4})-(?P<m>\d{2})", "2026-06")
m.group('y')   # '2026'
m.groupdict()  # {'y': '2026', 'm': '06'}

Character Classes

.Any character except newline (unless DOTALL flag).
\dDigit [0-9]. \D = non-digit.
\wWord char [a-zA-Z0-9_]. \W = non-word.
\sWhitespace. \S = non-whitespace.
[abc]Any one of a, b, or c.
[^abc]Any character except a, b, c.
[a-z]Range: any lowercase letter.
[a-zA-Z0-9_]Combined ranges (same as \w).

Anchors & Boundaries

^Start of string (or line, with MULTILINE).
$End of string (or line, with MULTILINE).
\AStart of string only (ignores MULTILINE).
\ZEnd of string only (ignores MULTILINE).
\bWord boundary.
\BNot a word boundary.

Quantifiers

*0 or more (greedy).
+1 or more (greedy).
?0 or 1 (greedy).
{n}Exactly n repetitions.
{n,}n or more.
{n,m}Between n and m, inclusive.
*? +? ?? {n,m}?Lazy / non-greedy versions — match as few as possible.
re.findall(r"<.+>", "<a><b>")   # ['<a><b>']  (greedy)
re.findall(r"<.+?>", "<a><b>")  # ['<a>', '<b>'] (lazy)

Groups & Alternation

(...)Capturing group — saves matched text, numbered left-to-right starting at 1.
(?:...)Non-capturing group — groups for quantifiers/alternation without saving a backreference. Cheaper and keeps group numbering clean.
(?P<name>...)Named capturing group — access via .group('name') instead of a number.
(?P=name)Backreference to a named group within the pattern itself (not in replacement text).
\1, \2, ...Backreference to numbered group, usable both inside the pattern and in re.sub replacement strings.
\g<name>Reference a named group inside a re.sub replacement string (safer than \1 when followed by digits).
a|b|cAlternation — match a, b, or c. Tries left to right, stops at first success.
(?:a|b)+Group alternation with a quantifier — repeat the whole "a or b" unit.
(a(b)c)Nested groups — numbered by order of the opening parenthesis (outer = 1, inner = 2 here).
(...)?Optional group — group may or may not appear; .group(n) is None if it didn't match.

Group numbering example — (\w+)-(?:(\d+))?-(\w+) has groups 1 and 3 only as "real" numbered captures if you don't count the non-capturing one; non-capturing groups never get a number at all.

## Basic capture + swap via backreference
re.sub(r"(\w+)@(\w+)", r"\2@\1", "user@domain")
# 'domain@user'

## Named groups, referenced by name in substitution
re.sub(r"(?P<first>\w+) (?P<last>\w+)", r"\g<last>, \g<first>", "Ada Lovelace")
# 'Lovelace, Ada'

## Non-capturing group used just for repetition
re.findall(r"(?:ab)+", "ababab xx ab")
# ['ababab', 'ab']

## Alternation choosing between formats
re.findall(r"\d{3}-\d{4}|\(\d{3}\)\d{3}-\d{4}", "call 555-1234 or (555)123-4567")
# ['555-1234', '(555)123-4567']

## Optional group: domain may or may not have a subdomain
m = re.match(r"(?:(\w+)\.)?example\.com", "api.example.com")
m.group(1)  # 'api'
m2 = re.match(r"(?:(\w+)\.)?example\.com", "example.com")
m2.group(1)  # None

## Repeated/nested capture only keeps the LAST iteration's match
re.findall(r"(\w)+", "abc")        # findall on repeated group only sees last char: ['c']
re.findall(r"(\w+)", "abc def")    # whole-word groups work as expected: ['abc', 'def']

Gotcha: a capturing group inside a * or + only retains the last repetition's match — use findall/finditer with the quantifier outside the group if you need every repetition.

Lookaround (Zero-Width Assertions)

(?=...)Positive lookahead — what follows must match ..., but it isn't consumed (doesn't appear in the match).
(?!...)Negative lookahead — what follows must NOT match ....
(?<=...)Positive lookbehind — what precedes must match ..., not consumed. Must be fixed-width in Python's re.
(?<!...)Negative lookbehind — what precedes must NOT match .... Also fixed-width only.
(?=a)(?=b)Stacked lookaheads — both conditions must hold at the same position (a kind of AND).

"Zero-width" means the assertion checks a condition but consumes no characters — the match position doesn't move forward or backward because of it.

## Positive lookahead: number only when followed by a unit
re.findall(r"\d+(?=px)", "10px 20em 30px")
# ['10', '30']

## Negative lookahead: word not followed by a colon
re.findall(r"\b\w+\b(?!:)", "key: value plain")
# matches 'value' and 'plain', skips 'key' (it's followed by ':')

## Positive lookbehind: amount only when preceded by '$'
re.findall(r"(?<=\$)\d+", "Cost: $50, Weight: 10kg")
# ['50']

## Negative lookbehind: number NOT preceded by '$'
re.findall(r"(?<!\$)\b\d+\b", "Cost: $50, Weight: 10")
# ['10']

## Password validation: stacked lookaheads (AND logic)
## Require: at least 8 chars, one digit, one uppercase letter
pattern = r"^(?=.*\d)(?=.*[A-Z]).{8,}$"
bool(re.match(pattern, "Secure123"))  # True
bool(re.match(pattern, "secure123"))  # False (no uppercase)

## Splitting on a comma NOT inside parentheses (lookahead trick)
re.split(r",(?![^(]*\))", "a,b,(c,d),e")
# ['a', 'b', '(c,d)', 'e']

## Lookbehind must be fixed width -- this raises an error:
## re.findall(r"(?<=\d+)px", "10px")   # error: variable-width lookbehind not allowed
## Workaround: use a fixed-width alternative or lookahead instead
re.findall(r"\d(?=px)", "10px")  # captures last digit before 'px' only: ['0']

Lookahead works with variable-width patterns (.*, +, etc.); lookbehind in Python's built-in re module requires a fixed width. The third-party regex module lifts that restriction if you need it.

Flags

re.IGNORECASE / re.ICase-insensitive matching.
re.MULTILINE / re.M^ and $ match start/end of each line.
re.DOTALL / re.S. matches newline too.
re.VERBOSE / re.XAllow whitespace/comments in pattern for readability.
re.ASCII / re.A\w \d \s match ASCII only, not Unicode.
re.search(r"hello", "Hello World", re.IGNORECASE)
re.compile(r"\d+", re.MULTILINE | re.IGNORECASE)

Common Escapes

\n \t \rNewline, tab, carriage return.
\\Literal backslash.
\.Literal period (escape regex metacharacters).
r"..."Raw string — always use for patterns to avoid double-escaping.

Special chars needing escape: . ^ $ * + ? { } [ ] \ | ( )

Practical Recipes

Email check 
re.match(r"[\w.+-]+@[\w-]+\.[a-zA-Z]{2,}", s)
Extract numbers 
re.findall(r"-?\d+\.?\d*", "Price: -3.5, Qty: 10")
Split on multiple delimiters 
re.split(r"[,;]\s*", "a, b; c,d")
Remove extra whitespace 
re.sub(r"\s+", " ", "too   many   spaces").strip()
Validate only-digits string 
bool(re.fullmatch(r"\d+", s))
Named groups for parsing 
re.match(r"(?P<key>\w+)=(?P<val>.+)", "name=Claude").groupdict()
