A Basic Guide to JSON

JavaScript Object Notation — a lightweight format for storing and exchanging data.

What is JSON?

JSON (JavaScript Object Notation) is a text format for representing structured data — things like configuration files, API responses, and saved settings. Despite the name, it isn't tied to JavaScript; nearly every programming language can read and write it.

It's built from just a handful of building blocks: objects, arrays, strings, numbers, booleans, and null. That's it — learn these six things and you know JSON.

Objects: key/value pairs

An object is wrapped in curly braces { }. Inside, every key is a string in double quotes, followed by a colon, then its value. Pairs are separated by commas. 

{
  "name": "Ada Lovelace",
  "age": 28,
  "isProgrammer": true
}
Keys must always be in double quotes — name: is invalid; it has to be "name":.

Arrays: ordered lists

An array is wrapped in square brackets [ ], with items separated by commas. Items can be any JSON type, including more arrays or objects. 

["red", "green", "blue"]

[1, 2, 3]

Putting it together

Real-world JSON usually nests objects and arrays inside each other. Here's a small example describing a person: 

{
  "name": "Grace Hopper",
  "birthYear": 1906,
  "isAlive": false,
  "languages": ["COBOL", "FORTRAN"],
  "address": {
    "city": "New York",
    "country": "USA"
  },
  "mentor": null
}

Here, "languages" is an array nested inside the outer object, and "address" is itself a whole object nested inside it. JSON can nest as deeply as you need.

Rules to remember

  • Double quotes only — strings and keys can't use single quotes ('like this' is invalid).
  • No trailing commas — ["a", "b",] with a comma after the last item is invalid.
  • No comments — JSON has no syntax for // or /* */ comments.
  • Keys are always strings — even if they look like numbers, they need quotes.
  • Commas separate, they don't terminate — only put a comma between items, never after the last one.

Common mistakes

❌ Invalid

{
  name: 'Sam',
  "age": 30,
}

Unquoted key, single-quoted string, trailing comma.

✅ Valid

{
  "name": "Sam",
  "age": 30
}

Quoted key, double-quoted string, no trailing comma.

Writing JSON in Python

Python's built-in json module converts between Python objects and JSON text. A Python dict maps naturally onto a JSON object. 

import json

data = {
    "name": "Ada Lovelace",
    "age": 28,
    "languages": ["Analytical Engine notes"]
}

# Convert Python -> JSON string
json_string = json.dumps(data, indent=2)
print(json_string)

# Write to a file
with open("person.json", "w") as f:
    json.dump(data, f, indent=2)

# Read it back
with open("person.json") as f:
    loaded = json.load(f)
dumps ("dump string") returns a JSON string. dump writes straight to a file. The load/loads pair work the same way in reverse, turning JSON back into Python objects.
