macOS Terminal — command cheatsheet

Basic–intermediate reference for the shell on macOS (zsh by default since Catalina)

Navigation

CommandEffect

Files & directories

CommandEffect
touch file.txtCreate an empty file, or update its modified time if it exists
mkdir folderCreate a new directory
mkdir -p a/b/cCreate nested directories, making parents as needed
cp src dstCopy a file
cp -r src dstCopy a directory recursively

rm fileDelete a file (no trash, no undo)
rm -r folderDelete a directory and its contents recursively
rm -rf folderSame, but force — skips confirmation prompts. Use with care.

rm does not move files to the Trash — it deletes them immediately and permanently. There's no built-in undo.

Permissions & ownership

CommandEffect
ls -lShow permissions in the leftmost column, e.g. -rwxr-xr-x
chmod +x fileMake a file executable
chmod 755 fileSet permissions numerically — owner rwx, group/others rx
chmod -R 755 folderApply permissions recursively to a directory and its contents
chown user fileChange the owner of a file
chown user:group fileChange both owner and group
sudo commandRun a command with administrator privileges (prompts for your password)

Permission digits are read/write/execute as bits: 7 = rwx, 6 = rw-, 5 = r-x, 4 = r--. The three digits in 755 are owner/group/others, in that order.

Environment variables

CommandEffect
echo $HOMEPrint the value of a variable — the $ tells the shell to resolve the name first
echo "$HOME"Same, but quoted — safer when the value might contain spaces
printenvList every environment variable currently set
printenv PATHPrint the value of a single specific variable
envSame as printenv with no arguments — list all env vars
export NAME=valueSet a variable and make it available to any commands/programs you run afterward
NAME=value commandSet a variable only for the duration of one command, without exporting it
unset NAMERemove a variable from the environment
echo $PATHShow the list of directories the shell searches for executables, colon-separated
export PATH="$HOME/bin:$PATH"Prepend a directory to PATH — order matters; earlier entries are checked first

A variable only needs the $ when you're reading its value. When setting it with export NAME=value, write the bare name with no $.

Variables set with export only last for the current terminal session. To make one permanent, add the export line to ~/.zshrc (the default shell config file on modern macOS) and either restart the terminal or run source ~/.zshrc.

$NAME vs ${NAME}

FormWhat it does
$NAMEExpands the variable. The shell reads letters, digits, and underscores after $ as the name, and stops at the first character that can't be part of one
${NAME}Same expansion, but the braces explicitly mark where the name starts and ends — anything right after } is treated as separate literal text
${NAME:-default}Expands to NAME's value, or to default if NAME is unset or empty
${NAME:?message}Expands to NAME's value, or prints an error and exits if it's unset
${#NAME}Length of NAME's value, in characters
${NAME/old/new}Expands to NAME's value with the first match of old replaced by new
${10}The tenth positional argument — needs braces, since $10 would parse as $1 followed by a literal 0

The two forms behave identically on their own — echo $HOME and echo ${HOME} print the same thing. The difference shows up when a variable butts up against more letters, digits, or underscores: echo $NAME_file reads NAME_file as one (probably unset) variable name, while echo ${NAME}_file correctly inserts NAME's value followed by the literal text _file.

Rule of thumb: use $NAME for simple, standalone references. Reach for ${NAME} when concatenating a variable directly against other text, or when using one of the :-, :?, #, / style operations above.

Shell config & startup files

FileWhen it runs
~/.zshrcLoaded for every new interactive zsh session — the usual place for aliases, PATH changes, and exported variables
~/.zprofileLoaded once at login, before .zshrc — used for things that should only run once per login
~/.bash_profile / ~/.bashrcEquivalent files if you're using bash instead of the zsh default
source ~/.zshrcRe-run a config file in the current session without opening a new terminal window
echo $SHELLShow which shell you're currently running
chsh -s /bin/zshChange your default login shell to zsh (or substitute another shell's path)

Quick examples

GoalCommand
Find your shell's PATH entries, one per lineecho $PATH | tr ':' '\n'
Check if a variable is set at allecho ${MY_VAR:-"not set"}
Permanently add a folder to PATHAdd export PATH="/new/folder:$PATH" to ~/.zshrc, then source ~/.zshrc
Find all files over 100MB in home folderfind ~ -size +100M
Count how many files are in a folderls -1 | wc -l
See what's using a specific portlsof -i :3000
Free up disk space estimatedf -h
See disk usage of a folder, summarizeddu -sh folder

Variable expansion syntax like ${VAR:-default} prints default only if VAR is unset or empty — handy for scripts that need a fallback value.
